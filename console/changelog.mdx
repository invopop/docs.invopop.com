<Update label="2026-04-01" description="GOBL v0.309.0" tags={["GOBL", "Release"]}>
  **GOBL core v0.309.0**

  Major update to the GOBL core library with improvements across multiple country regimes and the EN16931 European standard.

  - Added `tax.Note` field for exemption reasons (BT-120) in the EU EN16931 addon
  - Added tax point date support (`Point` field on `bill.Tax`) for issue, delivery, or payment dates
  - Added BR-32 validation requiring taxes on document-level discounts
  - Added NBS extension for Brazil NF-Se service classification codes
  - Poland: tax combos with non-Polish countries now normalized as outside scope
  - Spain TicketBAI: customer validation now only required for non-simplified invoices
  - Fixed rate `Since` date comparison to be inclusive
  - Fixed Greece myDATA exemption code mappings and panic on invoices without `bill.Tax`
  - Fixed Greece reduced island tax rate
</Update>

<Update label="2026-04-01" description="gobl.fatturapa v0.58.0–v0.60.0" tags={["Italy", "SDI"]}>
  **Italy FatturaPA converter updates**

  Several releases improving Italian e-invoicing conversion:

  - Added `DatiCassaPrevidenziale` (fund contribution) support for social security and pension fund charges
  - Added `AltriDatiGestionali INVCONT` for reverse charge N2.1 lines
  - Added line item period date mapping (`DataInizioPeriodo` / `DataFinePeriodo`)
  - Fixed division by zero edge case
</Update>

<Update label="2026-04-01" description="gobl.ubl v0.36.0–v0.39.0" tags={["UBL", "Peppol", "Format"]}>
  **UBL format converter updates**

  Multiple releases enhancing the OASIS UBL converter for Peppol and European e-invoicing:

  - Added Peppol Profile ID support
  - Added UBL parsing capabilities
  - Added line period date support for converting and parsing
  - Added BT-7 tax point date, BT-11/14 project and sales order references
  - Added BT-128/158/159 line and item identifiers
  - Added BT-90 bank assigned creditor identifier
  - Added `tax.Note` support for exemption reasons
  - Added French invoice context identification during parsing
  - Improved tax mappings and note parsing accuracy
</Update>
