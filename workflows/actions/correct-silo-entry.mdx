## Overview

Execute the correction procedure on the specified silo entry by applying the modifications in the provided JSON object. This ensures errors or inconsistencies in the original entry are patched. Once the corrections are applied, a new, revised version of the entry is generated and stored.

## Availability

* Invoice workflows
* Parties workflows
* Receipts workflows
* Rescue workflows

## Options

1. Correction options:

```
	{
	  "type": "credit-note",
	  "series": "",
	  "reason": ""
	}
```
