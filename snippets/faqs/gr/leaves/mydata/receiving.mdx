  <Accordion title="How do I import received invoices via myDATA?">
    Use IAPR's `RequestDocs` API to pull received invoices by AFM and MARK. Invopop's Greece app supports manual conversion of pulled XMLs into GOBL — automated polling is on the roadmap.
  </Accordion>
  <Accordion title="What's the current import workflow for Greek received invoices?">
    Invopop's Greece app does not yet offer automated received-invoice ingestion — manual import via the Convert action is the current path. Automated polling is on the roadmap.
  </Accordion>

  <Accordion title="What format do received myDATA invoices arrive in?">
    The `InvoicesDoc` XML schema published by IAPR — same shape as outbound submissions, with the issuer's signature and IAPR-assigned MARK and UID identifiers.
  </Accordion>
  <Accordion title="How does Invopop convert received Greek invoices into GOBL?">
    Inbound myDATA-format XMLs are parsed via the [`gobl.gr-mydata`](https://github.com/invopop/gobl) library. Tax categories, classification codes, and counterpart AFMs map directly onto GOBL fields.
  </Accordion>
