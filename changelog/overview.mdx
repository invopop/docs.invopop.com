<Update label="May 12, 2026" description="Weekly update" tags={["Saudi Arabia", "Spain", "Poland", "Email"]}>

## New features

**Bizkaia activity code for individual suppliers.** TicketBAI now supports the `es-tbai-bi-activity` extension, letting *autónomos* in Bizkaia declare their economic activity code (*epígrafe*) on invoices. The code is required for LROE Modelo 140 reporting and is set on the supplier party. See the [TicketBAI invoicing guide](/guides/es-ticketbai).

**Polish unit of measure on KSeF line items.** Line items submitted to KSeF can now populate the `P_8A` (*jednostka miary*) field two ways: with canonical UN/ECE unit codes via `Item.Unit` (e.g. `H87`, `KGM`) or with free-form Polish labels via the `unit-label` meta key (e.g. `szt.`, `usł.`, `kpl.`, `opak.`). Details in the [Poland FAQ](/faq/poland).

## Updates

**Saudi Arabia (ZATCA FATOORA) coverage announced.** Invopop will support Saudi Arabia's ZATCA e-invoicing mandate from **June 1st 2026**, covering both standard tax invoices (B2B/B2G clearance) and simplified tax invoices (B2C reporting), plus credit and debit notes. The compliance reference for Saudi Arabia has been published with the full FATOORA model — Phase 1 and Phase 2 obligations, mandatory invoice content, QR code TLV tags, VAT rates (including private education, private healthcare and qualified military goods as zero-rated), archival rules and the latest Wave 23–24 onboarding deadlines. See [Saudi Arabia compliance](/compliance/saudi-arabia). For early-access enquiries, email [support@invopop.com](mailto:support@invopop.com).

## Bug fixes

**Email recipients are now merged, not overwritten.** When the *Automatically add invoice customers* option is enabled, addresses manually entered in the `To` field are now combined with the auto-detected customer or party emails, so every intended recipient receives the message. Previously, explicit recipients could be overwritten by the auto-detected list. See the [Email guide](/guides/email).

</Update>

<Update label="May 10, 2026" description="Weekly update" tags={["France", "Mexico", "Italy"]}>

## New features

**France app.** The new France app centralises every French e-invoicing flow under the **Plateforme Agréée (PA)** model, including Annuaire registration, UBL/CII/Factur-X document generation, Peppol transport, and lifecycle status updates. Invopop is an officially approved Plateforme Agréée under the DGFiP mandate. See the [France app](/apps/france) and the [PA overview](/guides/fr-pa) to get started.

**Italy SDI B2G example.** The SDI Italy app now ships with a reference example for B2G (FatturaPA) invoices, alongside the existing B2B example. View it from the [SDI Italy app](/apps/sdi-italy) page.

## Updates

**Mexico CFDI compliance reference.** The Mexico compliance page has been rewritten with a clearer breakdown of CFDI document types, complementos, PAC clearance flow, and current Invopop coverage (including planned support for Pago receipts in Q2 2026). See [Mexico compliance](/compliance/mexico).

</Update>

<Update label="May 7, 2026" description="Weekly update" tags={["Spain", "France", "Portugal", "API"]}>

## New features

**NO VERI\*FACTU support for Spain.** You can now register VAT-exempt and out-of-scope invoices through SII without going through VERI\*FACTU. Useful for suppliers who report to AEAT directly but operate outside the VERI\*FACTU regime. Learn more in the [NO VERI\*FACTU supplier guide](/guides/es-noverifactu-supplier) and the [NO VERI\*FACTU invoicing guide](/guides/es-noverifactu).

**IEET (Catalonia tourist tax) examples.** VERI\*FACTU and SII now ship with reference examples covering the *Impost sobre les Estades en Establiments Turístics*, making it easier to issue compliant invoices that include the Catalonia tourist tax.

**Fetch all jobs endpoint.** A new endpoint lets you list every Transform job in a workspace, with filtering and pagination. See the [Fetch all jobs reference](/api-ref/transform/jobs/fetch-all-jobs).

## Updates

**France Plateforme Agréée (PA) guides refreshed.** The PA documentation has been streamlined and reorganized to better reflect the production rollout: clearer registration, invoicing, status, and reporting flows. Start at the [France PA overview](/guides/fr-pa).

**Updated workflow templates.** Console workflow templates have been refreshed with clearer condition handling and exception flows. See the [workflows guide](/guides/workflows) for how to use them.

**Chargebee integration improvements.** The Chargebee app's email behavior has been updated for more predictable invoice delivery. See the [Chargebee guide](/guides/chargebee).

## Removed

**InvoiceXpress integration retired.** The InvoiceXpress app for Portugal has been removed. Portuguese suppliers should use the [AT Portugal app](/apps/at-portugal) for SAF-T submission and series management going forward.

</Update>
