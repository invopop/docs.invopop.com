<Update label="June 19, 2026" description="Mexico digital platform reporting" tags={["Mexico"]}>

## New features

**Digital platform reporting for Mexico.** Operators of digital platforms ("plataformas digitales tecnológicas") can now meet the SAT's Article 30-B reporting obligation directly from Invopop. Three new actions — `sat-mx.platforms.register`, `sat-mx.platforms.record` and `sat-mx.platforms.unregister` — enrol a platform, record each operation with the fields required by rule 2.9.21 (service type, buyer RFC, amounts, VAT, folio fiscal, payment method), and produce a private, HMAC-signed portal URL that the SAT can use to browse, filter and export operations on demand. This is a separate obligation from CFDI issuance and works for both domestic and foreign operators. See the [platform reporting guide](/guides/mx-sat-platform-reporting) and [Mexico compliance](/compliance/mexico).

<Note>
  Digital platform reporting is in active development. Action names and portal layout may still change — please get in touch before enabling it in production.
</Note>

</Update>

<Update label="May 14, 2026" description="TicketBAI extensions" tags={["Spain"]}>

## New features

**Spain TicketBAI VAT regime key.** The new `es-tbai-regime` extension on `es-tbai-v1` sets the `ClaveRegimenIvaOpTrascendencia` code explicitly on a tax combo, covering all values from the TicketBAI XSD (`01`–`17`, `51`–`53`). GOBL keeps its previous defaults (export → `02`, equivalence surcharge → `51`, `simplified-scheme` tag → `52`, otherwise `01`) when the extension is not set. See the [TicketBAI invoicing guide](/guides/es-ticketbai#vat-regime-key).

**Spain TicketBAI customer identity type.** The new `es-tbai-identity-type` extension on `es-tbai-v1` records the L7 `IDType` code for customer identities reported under `IDOtro`, used when the customer has no tax ID. Common identity `key` values (`passport`, `foreign`, `resident`, `other`) are mapped automatically. See the [TicketBAI invoicing guide](/guides/es-ticketbai#customer-identity-type).

</Update>

<Update label="May 13, 2026" description="GOBL builder update" tags={["Argentina", "Spain"]}>

## New features

**Argentina tourism invoices (Factura T).** ARCA's Type T tourism invoice (WSCT) is now supported in the GOBL builder. Hotels and other tourism operators issuing invoices to foreign tourists can use the new `ar-arca-tourism-type` and `ar-arca-tourism-item` extensions on `ar-arca-v4` to flag the invoice and individual lines. See the [Argentina app](/apps/argentina) and the [ARCA invoicing guide](/guides/ar-arca-invoices).

**Spain IRNR tax category.** A new `IRNR` (Impuesto sobre la Renta de no Residentes) tax category is available for non-resident income tax withholdings. Rates depend on the type of income and recipient, so you supply them per invoice rather than relying on predefined rates. Useful for Spanish payors withholding tax on payments to non-resident suppliers.

**Spain TicketBAI Bizkaia activity code.** The new `es-tbai-bi-activity` extension on `es-tbai-v1` records the Bizkaia activity code (epígrafe) required for individual suppliers (autónomos) issuing through TicketBAI in Bizkaia. See the [TicketBAI invoicing guide](/guides/es-ticketbai).

**Spain FacturaE subsidy flag.** The new `es-facturae-subsidy` extension (`S` / `N`) on payment records identifies advance payments that come from a public grant or subsidy, matching the convention used in SII and VERI\*FACTU. See the [FacturaE app](/apps/facturae-spain).

## Updates

**Granular payment means.** Card payments now distinguish between credit (`card+credit`) and debit (`card+debit`) at the GOBL level, propagating to country-specific extension mappings where applicable.

**Multiple payment methods per payment document.** A single payment document can now record multiple methods with their own amounts and currencies, required for Portugal's SAF-T among others. See the [AT Portugal app](/apps/at-portugal).

</Update>

<Update label="May 12, 2026" description="Weekly update" tags={["Saudi Arabia"]}>

## Updates

**Saudi Arabia (ZATCA FATOORA) coverage announced.** Invopop will support Saudi Arabia's ZATCA e-invoicing mandate from **June 1st 2026**, covering both standard tax invoices (B2B/B2G clearance) and simplified tax invoices (B2C reporting), plus credit and debit notes. The compliance reference for Saudi Arabia has been published with the full FATOORA model — Phase 1 and Phase 2 obligations, mandatory invoice content, QR code TLV tags, VAT rates (including private education, private healthcare and qualified military goods as zero-rated), archival rules and the latest Wave 23–24 onboarding deadlines. See [Saudi Arabia compliance](/compliance/saudi-arabia). For early-access enquiries, email [support@invopop.com](mailto:support@invopop.com).

</Update>

<Update label="May 10, 2026" description="Weekly update" tags={["France", "Mexico", "Italy"]}>

## New features

**France app.** The new France app centralises every French e-invoicing flow under the **Plateforme Agréée (PA)** model, including Annuaire registration, UBL/CII/Factur-X document generation, Peppol transport, and lifecycle status updates. Invopop is an officially approved Plateforme Agréée under the DGFiP mandate. See the [France app](/apps/france) and the [PA overview](/guides/fr-pa) to get started.

**Italy SDI B2G example.** The SDI Italy app now ships with a reference example for B2G (FatturaPA) invoices, alongside the existing B2B example. View it from the [SDI Italy app](/apps/sdi-italy) page.

## Updates

**Mexico CFDI compliance reference.** The Mexico compliance page has been rewritten with a clearer breakdown of CFDI document types, complementos, PAC clearance flow, and current Invopop coverage (including planned support for Pago receipts in Q2 2026). See [Mexico compliance](/compliance/mexico).

</Update>

<Update label="May 7, 2026" description="Weekly update" tags={["Spain", "France", "Portugal", "API"]}>

## New features

**NO VERI\*FACTU support for Spain.** You can now register VAT-exempt and out-of-scope invoices through SII without going through VERI\*FACTU. Useful for suppliers who report to AEAT directly but operate outside the VERI\*FACTU regime. Learn more in the [NO VERI\*FACTU supplier guide](/guides/es-noverifactu-supplier) and the [NO VERI\*FACTU invoicing guide](/guides/es-noverifactu).

**IEET (Catalonia tourist tax) examples.** VERI\*FACTU and SII now ship with reference examples covering the *Impost sobre les Estades en Establiments Turístics*, making it easier to issue compliant invoices that include the Catalonia tourist tax.

**Fetch all jobs endpoint.** A new endpoint lets you list every Transform job in a workspace, with filtering and pagination. See the [Fetch all jobs reference](/api-ref/transform/jobs/fetch-all-jobs).

## Updates

**France Plateforme Agréée (PA) guides refreshed.** The PA documentation has been streamlined and reorganized to better reflect the production rollout: clearer registration, invoicing, status, and reporting flows. Start at the [France PA overview](/guides/fr-pa).

**Updated workflow templates.** Console workflow templates have been refreshed with clearer condition handling and exception flows. See the [workflows guide](/guides/workflows) for how to use them.

**Chargebee integration improvements.** The Chargebee app's email behavior has been updated for more predictable invoice delivery. See the [Chargebee guide](/guides/chargebee).

## Removed

**InvoiceXpress integration retired.** The InvoiceXpress app for Portugal has been removed. Portuguese suppliers should use the [AT Portugal app](/apps/at-portugal) for SAF-T submission and series management going forward.

</Update>
