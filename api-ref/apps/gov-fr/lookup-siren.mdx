---
title: "Look up directory entries by SIREN"
openapi: "GET /apps/gov-fr/v1/directory/siren/{siren}"
---

Returns the active directory entries from the local mirror of the PPF Annuaire that match the given SIREN. A SIREN may have multiple registered parties (one per SIRET / electronic address), so the response is always a list.

For each unique party (SIREN / SIRET / suffix / routing code), only the most recent directory line is considered, and it is included in the response only when its nature is **deposit** (`D`) — meaning the party is currently registered. Parties whose latest line is a **modification** (`M`, which cancels a previously deposited entry) are excluded.

An empty `results` array means the SIREN is not currently registered in the Annuaire and should be routed through the [e-reporting flow](/guides/fr-pa-reporting) instead of Peppol.

Only available to live workspaces — sandbox enrollments receive `403 Forbidden`.
